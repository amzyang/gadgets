# 上下文源卡片（context providers）

lark-watch 起草时求证/查证用的可插拔信息源。真实卡片放
`~/.config/lark-watch/context/*.md`（可能含私有路径，属用户数据，不进 git）；
本目录只放占位符模板，新机器初始化时复制过去按环境改。

## 格式

frontmatter 必填三字段：`provides`（能给什么信息）、`when`（何时找我）、
`cost`（fast/slow，slow 排后执行）。body 为只读命令指引，模型按实际话题
改写查询词——卡片是给模型读的说明书，不是给代码调的 API。

## 消费场景

表态门禁求证、反敷衍查证、洞察关联（见 SKILL.md 步骤 4/5）。目录缺失或
为空时模型静默回退到内置保底（拉长本会话历史），不报错。

## 隐私

卡片描述的源越具体越好用（具体仓库路径、wiki space、部门文档），这正是
它们只放 XDG 目录、不写进 SKILL.md 的原因。模板里的私有信息一律用
`<占位符>`。
